Add a new record:

Press the button that has the plus sign on it. Enter the part no. and description. Other record fields (dwg no., date, author) will be automatically filled in. If only the first four digits of a part no. are entered, then once the part no. is submitted the remainder of the part no. will be appended automatically. For example 0300 will turn into 0300-2020-401.

After you enter the first four digits for a part number, if the number is for a common part type, the description field will fill in automatically with a partial description. Finish filling in the description field or alter it accordingly. For example, entering 0300 in the pt. no. field will cause the word BASEPLATE to show in the description field.

Search the database:

To search, enter a query into the box provided and push the Enter key. Here are some examples:

  example 1, query:

  • 2730-2020-8*

  yields:

  • Dwg. no.Part no.DescriptionDate Author
    20208702730-2020-870BRACKET PANEL 20X20 200GAL TANK CS12/04/2020tasbury
    20208632730-2020-863PANEL BRACKET 16x14 CS12/02/2020whoyt
    etc.

  example 2, query:

  • 1[12]/*/2020; kcarlton

  yields:

  • Dwg. no.Part no.DescriptionDate Author
    20208590300-2020-859BASEPLATE FOR DV0016B-DA2 W/MONO BLK MTR12/01/2020kcarlton
    2020857094411DV0016B-DA2 W/MONO BLK MTR & BASE11/27/2020kcarlton
    etc.

Here found are records (a record being a row of data) where kcarlton is the author, and the date of creation is November or December of 2020. The [12] means 1 or 2, thus 1[12] means months 11 or 12. The * in 1[12]/*/2020 means any day of the month.

There are actually two query groups that are searched for in the above example: 1[12]/*/2020 and kcarlton. The semicolon, ;, acts as a special delimiter character in the program. The semicolon results in the program only displaying results that each query group has in common. Put another way, the semicolon finds the intersection of results from the two query groups.

  example 3, query:

  • 09*; kcar*; 11/*/2020   or   09*; rcol*; 11/*/2020   or   09*; wh*; 11/*/2020

  yields:

  • Dwg. no.Part no.DescriptionDate Author
    2020857094411DV0016B-DA2 W/MONO BLK MTR & BASE11/27/2020kcarlton
    2020855094379DVW0303KP1-35 230/460V TEFC 11/25/2020rcollins
    2020823094020VMX0753KA1-00 460V TEFC 11/13/2020whoyt
    etc.

The groups in the query with or without the semicolon character are searched first. The "or" keyword that separates those groups acts to find any or all of the results from those individual groups. The word "or" must be in lower case letters.

Note that searches are case sensitive. That is *rc* and *RC* will yield different results. For more information about searching, see: https://en.wikipedia.org/wiki/Glob_(programming)

Update a field:

To update a field (a table cell), change the text in the field and then press the Enter key.

If in a part no. field you put the first four digits and a dash (e.g. 0300-), the part no. will autogenerate (e.g. 0300-2021-030).

A drawing number may be changed. For example, you can change drawing number 2020456 to 2020460 if the number 2020460 is available. If the number is unvailable, the program will show an error message and the change will not occur.

When an drawing number is changed, the part no. will attempt to change automatically. For example, if for drawing number 2020456 the part number is 0300-2020-456, changing the drawing number to 2020460 will change the part no. to 0300-2020-460.

Only if company policy permits, a drawing number may be overwritten. A drawing number looking like 2020466 is autogenerated by the program. If the user changes the drawing number to something like 3201T112 or AC20466 (for Atlas Copco), both of which are in a nonstandard format, the original autogenerated drawing number will be overwritten. To retrieve the original number, edit the drawing no. field, leave it black, and then press Enter.

Delete a record:

To delete a data record (a table row), enter one of the following words into the field that contains the drawing number: delete, remove, del, or rm. Then press Enter.

Refresh the table:

While you are working on the dwglog2 program, other users simultaneously have access to the same data shown to you. You may wish to push the refresh button to see changes other users have made while you have been working with the program. Also, if you wish to assure that your data has been updated successfully, push the refresh button. The most recent data will be pulled from the database.

AutoCopy:

Automatically copy contents of a clicked field (cell) to the clipboard. That is, Ctrl-C not needed.
